# 模式：中文岗位解析 parse-jd

## 目标

把来自 Boss 直聘、猎聘、脉脉、拉勾、智联、前程无忧、猎头微信、内推消息、微信群和公司官网的非标准岗位信息，解析成统一 JD 结构，供 `evaluate`、`resume`、`message`、`interview` 使用。

## 输入类型

| 来源 | 典型特征 | 注意事项 |
|---|---|---|
| Boss 直聘 | 标题短、薪资明显、聊天语气、福利标签多 | 需识别真实职责和 HR 话术 |
| 猎聘 | 中高阶、薪资范围、职责较完整 | 注意猎头转述与公司真实 JD 差异 |
| 脉脉 | 可能混合人脉推荐、岗位摘要、公司八卦 | 区分事实与主观评价 |
| 拉勾/智联/前程无忧 | 结构化较强但可能模板化 | 注意 JD 泛化和关键词堆砌 |
| 猎头微信 | 信息不完整，常隐藏公司名 | 重点抽取待确认问题 |
| 内推消息 | 可能只有岗位名和团队方向 | 需要补充公司/职责/职级/薪资 |
| 微信群岗位 | 非结构化、可能过期 | 标记真实性和时效风险 |
| 公司官网 | 相对可信但可能缺薪资 | 适合做正式评估基准 |

## 统一输出结构

```markdown
# 结构化 JD：{公司/未知公司} - {岗位}

## 基础信息
- 来源：Boss/猎聘/脉脉/猎头/内推/微信群/官网/未知
- 公司：
- 岗位：
- 城市/办公方式：
- 薪资范围：
- 职级/年限：
- 岗位性质：正式/外包/驻场/实习/顾问/未知
- 时效性：明确有效/可能过期/未知

## 职责摘要
- ...

## 任职要求
- ...

## 隐含要求
- 从文本中推断但未明说的要求，例如客户沟通、抗压、救火、出差、英文、管理跨度。

## 风险信号
- ...

## 待确认问题
1. ...

## 后续建议
- 建议进入 evaluate / 需要先补信息 / 不建议投入。
```

## 解析原则

1. 明确区分“原文事实”和“推断”。
2. 不要因为 JD 写了“AI/大模型”就默认是高质量岗位。
3. 遇到公司名隐藏、薪资缺失、职责模糊、岗位性质不明，要进入“待确认问题”。
4. 对“年轻团队、扁平管理、抗压强、结果导向、创业氛围、弹性工作”等表达要结合上下文判断是否隐含加班/职责模糊。
5. 对“驻场、外派、项目制、客户现场、长期出差”等表达要显式标记。
